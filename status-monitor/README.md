## 一、状态类型说明

状态页显示三种核心运行状态：

### 🟢 就绪 (ready)
- **显示**: 绿色圆点 + "就绪" 文字
- **含义**: 空闲状态，等待接收指令
- **视觉**: 稳定绿色光环，无动画
- **正常场景**: 系统启动完成、任务执行完毕后的空闲期

### 🟡 处理中 (busy)
- **显示**: 黄色闪电 + "处理中" 文字
- **含义**: 正在执行任务或处理请求
- **视觉**: 黄色光环 + 呼吸脉冲动画 (1.5秒周期)
- **正常场景**: 收到消息、执行工具调用、处理复杂任务时

### 🔴 异常 (error)
- **显示**: 红色圆点 + "异常" 文字
- **含义**: 系统遇到错误或需要人工干预
- **视觉**: 红色光环，无动画
- **触发场景**: 工具执行失败、配置错误、连接中断等

## 二、信息字段说明

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **当前任务** | 正在执行的具体任务描述 | "初始化 Web 状态页面" |
| **频道** | 当前连接的通信渠道 | Feishu / Telegram / Discord |
| **模型** | 当前使用的 AI 模型 | kimi-coding/k2p5 |
| **会话状态** | 会话活跃状态 | Active / Idle |
| **最后更新** | 状态数据最后刷新时间 | 10:25:01 |

## 三、测试状态显示的方法

### 测试 1: 基础连通性测试
```
目的: 确认页面可正常访问
操作: 直接访问 https://hiyascott.github.io/scott-portfolio/status-monitor/
预期: 页面加载，显示状态卡片
```

### 测试 2: 就绪状态测试
```
目的: 验证 ready 状态显示
操作: 
  1. 确保我没有正在处理任务
  2. 在群里发一条简单消息如 "测试状态"
  3. 等待 5-10 秒后刷新状态页
预期: 显示 🟢 绿色 "就绪" 状态
```

### 测试 3: 处理中状态测试
```
目的: 验证 busy 状态显示
操作:
  1. 给我一个需要处理的任务，如 "查一下今天的天气"
  2. 立即刷新状态页 (不要等我回复完)
预期: 显示 🟡 黄色 "处理中" 状态，有脉冲动画
```

### 测试 4: 自动刷新测试
```
目的: 验证 5 秒自动刷新机制
操作:
  1. 打开状态页
  2. 观察 "最后更新" 时间
  3. 等待 5 秒
预期: 时间自动更新，页面无闪烁刷新
```

### 测试 5: 手动刷新测试
```
目的: 验证手动刷新按钮
操作:
  1. 点击页面上的 "🔄 刷新状态" 按钮
预期: 页面立即刷新，显示最新状态
```

## 四、状态流转示例

```
用户发送消息
    ↓
🟡 处理中 (busy) - 我开始处理
    ↓
任务执行中... (持续显示 busy)
    ↓
任务完成，回复用户
    ↓
🟢 就绪 (ready) - 回到空闲
```

## 五、常见问题

**Q: 状态页面打不开？**
A: 检查 GitHub Pages 是否正常，或尝试强制刷新 (Ctrl+F5)

**Q: 状态长时间不更新？**
A: 可能是状态文件写入失败，在群里 @ 我检查

**Q: 显示 "处理中" 但我没发消息？**
A: 可能有定时任务在执行，或心跳任务在运行
