## 二、根因分析

### 1. 天赋描述错误（最严重）

**问题表现**：
- 天赋描述过于简化，遗漏关键机制
- 部分天赋名称与实际不符
- 未标注星级差异（Wiki数据为6星状态）

**典型案例**：

| 角色 | 原描述 | 正确描述 | 遗漏信息 |
|------|--------|----------|----------|
| 冰璃 | 击杀后再行动 | 2层「怒意」后获得再行动(5格) | 怒意层数、属性加成、CD间隔 |
| 曹沁 | 击杀后刷新CD | 「流火」状态+闪避也可刷新 | 流火效果、触发条件 |
| 林月如 | 20%概率再行动 | 「气劲」4层自动眩晕 | 气劲机制、移动增伤 |

**根因**：
- 原始数据来源于早期攻略网站，非官方Wiki
- 攻略作者简化描述便于理解，但丢失细节
- 数据录入时未核对官方来源

---

### 2. 技能名称与效果错位

**问题表现**：
- 技能名称和描述来自不同版本/不同角色
- 天赋和技能混淆

**典型案例**：林月如
- 原天赋「斩龙诀」实际是技能名
- 原技能「斩龙破」「御剑术」在游戏中不存在
- 遗漏核心技能「一阳指」「乾坤一掷」

**根因**：
- 数据收集时使用了非官方来源（如玩家自制攻略）
- 未进行多源交叉验证
- 联动角色（仙剑）数据可能来自测试服/预览版本

---

### 3. 属性/品阶/职业错误

**问题表现**：

| 角色 | 错误 | 正确 | 可能原因 |
|------|------|------|----------|
| 燕明蓉 | 极 | 绝 | 早期版本为极，后升绝 |
| 鲜于超 | 炎 | 暗 | 属性符号混淆（炎/暗视觉相似）|
| 巴艾迩 | 羽士 | 御风 | 职业理解偏差（远程御风）|

**根因**：
- 游戏版本更新后数据未同步
- 人工录入时的视觉/理解错误
- 缺乏数据验证流程

## 四、数据采集经验教训

### ❌ 错误做法（本次踩坑）

1. **单一来源依赖**
   - 仅使用攻略网站，未核对官方Wiki
   - 攻略作者简化导致信息丢失

2. **缺乏版本控制**
   - 未标注数据对应的游戏版本
   - 角色升阶（极→绝）未同步

3. **未建立验证流程**
   - 数据录入后未抽样验证
   - 缺乏与官方数据的交叉比对

4. **技能与天赋混淆**
   - 数据结构不清晰
   - 天赋、技能、大招边界模糊

---

### ✅ 正确做法（后续改进）

#### 1. 数据源优先级

| 优先级 | 来源 | 可靠性 | 使用建议 |
|--------|------|--------|----------|
| P0 | 官方Wiki (B站) | ⭐⭐⭐⭐⭐ | 主要数据源 |
| P1 | 游戏内数据 | ⭐⭐⭐⭐⭐ | 最终验证 |
| P2 | 官方公众号/微博 | ⭐⭐⭐⭐ | 新角色预览 |
| P3 | 知名攻略作者 | ⭐⭐⭐ | 辅助参考 |
| P4 | 玩家论坛 | ⭐⭐ | 需谨慎验证 |

#### 2. 数据录入流程

```
数据获取 → 多源交叉验证 → 格式化录入 → 抽样核查 → 定期同步
```

**关键检查点**：
- [ ] 天赋描述是否包含完整机制
- [ ] 技能名称是否与官方一致
- [ ] 属性/品阶/职业是否匹配Wiki
- [ ] 是否有遗漏技能

#### 3. 数据结构设计

```json
{
  "hero": {
    "name": "角色名",
    "tier": "品阶",
    "element": "属相",
    "class": "职业",
    "talent": {
      "name": "天赋名",
      "description": "完整描述(6星)",
      "source_url": "Wiki链接"
    },
    "skills": [
      {
        "type": "主动/被动/大招",
        "name": "技能名",
        "description": "效果描述",
        "cd": "冷却",
        "range": "射程"
      }
    ],
    "tags": ["机制标签"],
    "last_verified": "2026-03-17",
    "data_source": "B站Wiki"
  }
}
```

#### 4. 定期维护机制

- **月度检查**: 新角色数据补充
- **季度核查**: 与Wiki同步，检查升阶/调整
- **年度重构**: 全面数据质量评估
