# 冈仁波齐灌溉系统 - 版本变更记录

> 遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/) 规范
> 
> 版本格式：主版本号.次版本号.修订号（MAJOR.MINOR.PATCH）
> - MAJOR：不兼容的 API 修改
> - MINOR：向下兼容的功能新增
> - PATCH：向下兼容的问题修正

## [1.6.0] - 2026-04-06

### Changed
- **重构灌溉与施肥管理模块边界**
  - 灌溉管理页面移除施肥阀控制模块
  - 灌溉管理仅保留：水泵、灌溉主阀、田间阀三级控制
  - 施肥阀控制完全迁移至施肥管理页面
- **优化页面结构**
  - 灌溉管理底部添加提示：施肥阀控制已移至施肥管理
  - 施肥管理页面顶部新增施肥阀控制卡片区域
  - 施肥阀支持独立开关和一键全部开启/关闭

### Added
- 施肥管理页面新增施肥阀快捷操作
  - 全部开启按钮（带确认弹窗）
  - 全部关闭按钮（带确认弹窗）
  - 操作日志自动记录

## [1.5.0] - 2026-04-06

### Added
- 新增灌溉程序管理功能
  - 可执行程序列表展示（启用/暂停状态）
  - 程序创建向导：选择阀门 → 选择作物 → 灌溉方式 → 频率 → 时间 → 时长
  - 支持滴灌/喷灌两种方式
  - 浇水频率设置：1-7天
  - 运行时长设置：1分钟-5小时（滑块控制）
- 新增时间冲突检测系统
  - 实时检测新程序与现有程序的时间重叠
  - 冲突时显示具体冲突的程序列表
  - 要求修改时间后才能保存
  - 启用程序时二次冲突检查
- 新增程序操作功能
  - 启用/暂停程序
  - 立即执行程序
  - 删除程序（带确认弹窗）
- 新增作物类型选项（青稞、油菜、马铃薯、小麦、大白菜等）

### Changed
- 优化灌溉管理页面布局
  - 新增灌溉程序卡片区域
  - 优化按钮分组和操作区域
- 灌溉程序执行时自动记录用水量到日志

## Commit Message 规范

### 格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type 说明
- `feat`: 新功能（feature）
- `fix`: 修复bug
- `docs`: 文档（documentation）
- `style`: 格式（不影响代码运行的变动）
- `refactor`: 重构（既不是新增功能，也不是修改bug）
- `perf`: 性能优化
- `test`: 增加测试
- `chore`: 构建过程或辅助工具的变动

### 示例
```
feat(irrigation): 添加水流量传感器监控

- 总水表实时流量显示
- 8个分区水表独立监控
- 3路肥水表按肥料类型分类

Closes #123
```

```
fix(ui): 修复移动端侧边栏遮罩问题

在iOS Safari上点击遮罩层无法关闭侧边栏
```

```
docs(readme): 更新部署说明

添加GitHub Pages部署步骤和注意事项
```
