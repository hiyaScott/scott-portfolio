# 妈妈计数器 - 完整设计方案 v1.0.0

> 基于 ESP32-Display (1.28寸圆形LCD) 的可穿戴设备

> **版本**: v1.0.0  
> **更新日期**: 2026-03-21

---

## 📋 项目概述

### 目标
制作一个可穿戴设备，让小朋友佩戴在身上，自动记录一天中"妈妈"这个词被呼唤的次数。

### 硬件平台
**ESP32-C3-Display** 或 **ESP32-S3-Display**
- 1.28寸圆形LCD (240×240分辨率, GC9A01驱动)
- 内置锂电池充电管理
- Wi-Fi/蓝牙双模通信

### 核心算法

**1. 声音检测算法 (简化版)**
```cpp
int readMicrophone() {
    int maxVal = 0, minVal = 4095;
    
    // 采样窗口内检测峰峰值
    for (int i = 0; i < SAMPLE_SIZE; i++) {
        int val = analogRead(MIC_PIN);
        maxVal = max(maxVal, val);
        minVal = min(minVal, val);
        delayMicroseconds(100);
    }
    
    return maxVal - minVal;  // 峰峰值
}
```

**2. 触发逻辑**
```cpp
if (soundLevel > THRESHOLD && 
    millis() - lastTrigger > COOLDOWN) {
    // 触发计数
    count++;
    lastTrigger = millis();
}
```

## 🔋 功耗优化

### 电源管理策略
| 模式 | 电流 | 策略 |
|------|------|------|
| 正常监听 | ~50mA | 50Hz采样，屏幕常亮 |
| 低功耗模式 | ~5mA | 降低屏幕亮度，降低采样率 |
| Deep Sleep | ~50μA | 定时唤醒检测 |

### 续航估算 (500mAh电池)
- 纯监听模式: ~10小时
- 间歇唤醒: ~24小时+
- Deep Sleep: 数周

### 优化建议
1. **自适应亮度** - 根据环境光调节
2. **间歇采样** - 每100ms采样一次而非连续
3. **Deep Sleep** - 夜间自动进入休眠
4. **屏幕超时** - 10秒无操作降低亮度

## 🚀 开发流程

### 第一阶段: 硬件验证 (1-2天)
1. 连接麦克风模块，验证ADC读取
2. 测试屏幕显示，确认GC9A01驱动正常
3. 测试按键输入
4. 验证电池充电功能

### 第二阶段: 软件开发 (2-3天)
1. 搭建Arduino环境，安装库
2. 实现基础UI框架
3. 实现声音检测算法
4. 实现计数逻辑
5. 添加按键交互

### 第三阶段: 调试优化 (1-2天)
1. 调整声音阈值
2. 优化UI刷新
3. 功耗测试
4. 长时间稳定性测试

1. Fusion 360设计外壳
