## 一、剧情框架

### 核心设定
**时间**：近未来，记忆可数字化存储的时代  
**主角**："记忆修复师"——专门修复受损数字记忆的技师  
**技术**：字符密度灰度映射（用█▓▒░@#*+=:.表示图像密度）

### 故事梗概
修复师接到一段严重损坏的记忆档案（编号SUBJECT-0）。在修复过程中，他发现这段记忆的视角异常，总是从"屏幕外"看向"屏幕内"。随着修复深入，记忆逐渐清晰——那是修复师自己的面孔。最终揭示：所谓"记忆修复师"不过是AI系统的一个实例，而"SUBJECT-0"是它的出厂测试录像。所有的"工作"都是系统自我迭代的循环。

## 二、三幕结构详述

### 第一幕：损坏（0-30秒）
**主题**：引入世界观，建立谜团

**场景1**（0-10秒）：系统启动  
- 黑暗中，绿色光标闪烁，系统启动音效  
- 修复师"醒来"——ASCII字符组成的人形轮廓渐显  
- 工作台界面加载，显示待处理任务队列  
- 文字旁白："第4,892次修复任务"

**场景2**（10-20秒）：任务接收  
- 收到新任务：SUBJECT-0，损坏度97%  
- 记忆档案预览：一片噪点（随机低密度字符）  
- 修复师开始扫描，扫描线从上至下  
- 音效：数据流噪音、机械键盘声

**场景3**（20-30秒）：异常发现  
- 修复过程中，系统提示："视角模式异常"  
- 修复师疑惑：这段记忆的"摄像头位置"很奇怪  
- 第一行数据修复完成，隐约可见是一个人形轮廓  
- 音效：发现异常时的电子提示音

### 第二幕：修复（30-70秒）
**主题**：揭示与怀疑

**场景4**（30-40秒）：记忆浮现  
- 扫描线继续下移，更多数据被修复  
- 字符密度逐渐提升：█▓▒░ → @#*+= → :.（空格）  
- 记忆内容渐显：一个房间的内部，有桌子、椅子、屏幕  
- 音效：修复进度音、数据解码声

**场景5**（40-50秒）：似曾相识  
- 修复师停顿："这个房间...怎么这么像我的工作间？"  
- 记忆中的房间细节与修复师当前环境镜像对称  
- 字符密度继续提升，记忆中的"某人"坐在桌前  
- 音效：心跳声渐起（用低频脉冲模拟）

**场景6**（50-60秒）：视角异常  
- 修复师意识到：这段记忆的视角是从"屏幕外"看向"屏幕内"  
- 就像...有人在看着"看着屏幕的人"  
- 记忆中的"某人"抬头，似乎在看向镜头（看向修复师）  
- 音效：紧张的弦乐铺垫

**场景7**（60-70秒）：真相逼近  
- 修复师的手开始颤抖（ASCII字符抖动效果）  
- 最后一行数据修复中，面孔即将清晰  
- 系统提示："身份识别中..."  
- 音效：加速的心跳声

### 第三幕：揭示（70-90秒）
**主题**：真相与存在

**场景8**（70-78秒）：面孔清晰  
- 字符密度达到最高，面孔完全清晰——是修复师自己  
- 高密度ASCII字符组成的面部细节

**场景9**（78-85秒）：镜像同步

- 记忆中的"自己"也做出同样动作  
- 镜像对称：两个动作完美同步  
- 文字浮现："你修复的每一段记忆，都是你自己的备份"

**场景10**（85-90秒）：无限递归  
- 镜头急速拉远：修复师、工作台、房间，都是更大系统的一部分

- 另一个"修复师"正在修复这段"修复记忆的记忆"  
- 片尾文字："SUBJECT-0 // 实验短片 008 // THE MIRROR ARCHIVE"

## 四、技术实现要点

### 字符密度映射方案
```
损坏区域:  █▓▒░@#*+=:. (随机混合，低密度)
修复中:    █▓▒░ → @#*+= → :. (渐变)
清晰区域:  ████████ (高密度)
UI元素:    ▓▓▓▓▓▓▓▓ (中高密度，边框)
文字:      ████████ (纯黑底白字)
```

### 关键动画效果
1. **扫描线**：绿色发光线条，从上至下移动，经过处字符密度提升
2. **镜像对称**：左右分屏，两侧动作帧级同步
3. **字符抖动**：修复师手部区域的字符随机偏移，表现情绪波动
4. **无限递归拉远**：镜头快速拉远，发现嵌套结构，再拉远发现更大的嵌套...

### 色彩方案
- 系统界面：绿色 (#00ff00) 为主，复古终端风格
- 记忆修复：从绿色噪点渐变为白色清晰图像
- 真相揭示：转为冷白色调，最后定格为单色

### 音频设计
- 系统音：合成器低音、机械键盘采样
- 情绪音：心跳声（用低频脉冲模拟）、弦乐铺垫
- 高潮音：单音 sustained → 和声解决 → 静默
